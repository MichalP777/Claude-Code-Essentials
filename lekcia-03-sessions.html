Sessions a
workflow manažment

Claude Code si pamätá kontext v rámci session. Nauč sa, ako sessions spravovať — obnoviť, vetviť, skrátiť, premenovať alebo vrátiť späť.

// čo sa naučíš

  • Pochopiť, čo je session a ako Claude Code uchováva kontext
  • Obnoviť prerušenú session príkazom --resume
  • Vetviť session príkazom --fork na experimentovanie
  • Spravovať históriu: compact, clear, rename, rewind
  • Sledovať stav a spotrebu: status, usage, ccusage
01

Čo je session?

Session je konverzačný kontext medzi tebou a Claude Code agentom. Obsahuje históriu správ, vykonané nástroje, prečítané súbory a rozhodnutia agenta. Každá session má unikátne session ID.

Keď zatvoríš terminál, session sa neuloží do pamäte — ale Claude Code si ju zapamätá na disku, takže sa k nej môžeš vrátiť.

claude spustený
→
session vytvorená
→
práca / tool calls
↓   zatvoríš terminál
session uložená
→
--resume → pokračuješ
|
--fork → nová vetva
// kde sú sessions uložené

Sessions sa ukladajú lokálne — zvyčajne v ~/.claude/sessions/. Neodchádzajú na server, sú len na tvojom stroji.

02

Resume a Fork — obnovenie a vetvenie

Sú to dva najdôležitejšie príkazy pre prácu so sessions. Rozdiel je kľúčový: resume pokračuje v tej istej session, fork vytvorí jej kópiu a nechá originál netknutý.

resume — pokračovanie
  • Vráti sa do existujúcej session
  • Kontext zostáva rovnaký
  • Ideálne pre prerušenú prácu
  • Jedna session, žiadna kópia
fork — vetvenie
  • Vytvorí novú kópiu session
  • Originál zostáva nedotknutý
  • Ideálne pre experimentovanie
  • Dve nezávislé session vetvy
resume a fork
# Zobraziť zoznam posledných sessions
$claude --list-sessions
abc123 2026-03-27 14:32 "refaktoring auth modulu"
def456 2026-03-26 09:15 "nový feature: dashboard"
# Obnoviť poslednú session
$claude --resume
✓ Resuming session abc123 — "refaktoring auth modulu"
# Obnoviť konkrétnu session podľa ID
$claude --resume def456
# Vetviť session — experimentálna vetva
$claude --fork abc123
✓ Forked session abc123 → new session ghi789
// kedy použiť fork

Fork je ideálny, keď chceš vyskúšať rizikovú zmenu (napr. prepísanie modulu) bez toho, aby si stratil fungujúci kontext. Ak experiment zlyhá, vrátiš sa k originálu.

03

Správa histórie — compact, clear, rename, rewind

Dlhé session konzumujú viac kontextového okna a môžu spomaľovať model. Tieto príkazy ti pomôžu udržať session čistú a efektívnu.

/compact
Skráti históriu session — zachová kľúčové informácie, odstráni nepodstatné detaily. Šetrí tokeny.
/clear
Vymaže celú históriu konverzácie a začne čistú session. Projekt a súbory zostanú nedotknuté.
/rename
Premenuje aktuálnu session na zmysluplný názov, aby si ju ľahšie našiel v zozname.
/rewind
Vráti session do predchádzajúceho stavu — ako Ctrl+Z pre celú konverzáciu.
správa histórie
# Skrátenie dlhej session (zachová kontext)
> /compact
✓ Session compacted: 12 400 → 3 200 tokens
# Premenovanie session
> /rename auth-refactor-sprint3
✓ Session renamed to "auth-refactor-sprint3"
# Vrátenie posledného kroku späť
> /rewind
✓ Rewound to previous state
# Vymazanie histórie (nová čistá session)
> /clear
⚠ Conversation history cleared.
// pozor — clear je nenávratné

/clear vymaže celú históriu konverzácie a nedá sa vrátiť. Ak si nie si istý, radšej použi /compact alebo --fork pred tým.

04

Monitorovanie — status, usage, ccusage

Sledovanie stavu a nákladov je dôležité pri dlhých agentných úlohách. Claude Code poskytuje niekoľko nástrojov na monitoring.

monitoring
# Aktuálny stav session a agenta
$claude status
✓ Logged in: michal@example.com
Model: claude-sonnet-4-6
Session: auth-refactor-sprint3 (abc123)
Context: 8 240 / 200 000 tokens
# Spotreba tokenov v aktuálnej session
> /cost
Input tokens: 8 240
Output tokens: 2 100
Estimated cost: ~$0.04
# Externý nástroj: podrobná história nákladov
$npx ccusage
Today: $0.18 (4 sessions)
This week: $1.24 (23 sessions)
This month: $4.87
// ccusage — externý nástroj

ccusage je open-source nástroj na sledovanie nákladov Claude Code. Inštalácia: npm install -g ccusage. Číta lokálne logy a zobrazuje prehľad spotreby.

05

Context commands — práca s kontextom

Okrem správy session môžeš priamo ovládať, čo agent vidí vo svojom kontexte. Tieto príkazy sú užitočné pri veľkých projektoch.

context management
# Zobraziť, čo agent aktuálne vidí v kontexte
> /context
Files in context: src/auth.ts, src/middleware.ts
Tools used: Read (×12), Edit (×4), Bash (×2)
# Pridať súbor do kontextu manuálne
> Prečítaj si aj src/config.ts
# Odhlásenie (ukončenie auth tokenu)
$claude logout
✓ Logged out successfully
// limit kontextu

Claude Sonnet má kontextové okno 200 000 tokenov. Sleduj /cost a keď sa blížiš k limitu, použi /compact alebo začni novú session cez --fork.

// rýchly test

Chceš vyskúšať riskantnú zmenu architektúry, ale nechceš prísť o aktuálnu session. Ktorý príkaz použiješ?

A /clear — vymaže históriu a začneš odznova
B claude --resume — obnoví poslednú session
C claude --fork — vytvorí kópiu session, originál ostane netknutý
D /compact — skráti históriu session
// zhrnutie lekcie 03

Sessions sú srdcom workflow manažmentu v Claude Code. Použi --resume na pokračovanie a --fork na bezpečné experimentovanie. Históriu udržiavaj čistú cez /compact a sleduj náklady pomocou /cost alebo ccusage.
