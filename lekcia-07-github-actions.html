GitHub Actions
a CI/CD

Claude Code nie je len interaktívny nástroj — beží aj bez teba. Nauč sa integrovať agenta do GitHub Actions pipeline a automatizovať kódovacie úlohy pri každom pull requeste.

// čo sa naučíš

  • Pochopiť headless mód — ako Claude Code beží bez interakcie
  • Nastaviť GitHub Actions workflow s Claude Code agentom
  • Bezpečne uložiť API kľúč ako GitHub Secret
  • Spúšťať agenta pri PR, push aj manuálne cez workflow_dispatch
  • Poznať typické use cases: code review, generovanie testov, dokumentácia
01

Headless mód — Claude Code bez interakcie

V predvolenom móde Claude Code čaká na tvoje správy interaktívne. Headless mód umožňuje spustiť agenta s konkrétnou úlohou, vykonať ju a skončiť — ideálne pre skripty a CI/CD pipeline.

headless mód
# Interaktívny mód (predvolený) — čaká na vstup
$claude
> _
# Headless — vykoná úlohu a skončí
$claude --headless "skontroluj testy a oprav failing prípady"
Running tests... 3 failures found
Fixing: test/auth.test.ts (line 42)
Fixing: test/api.test.ts (line 88)
✓ All tests passing. Exit 0.
# Skrátená forma
$claude -H "vygeneruj JSDoc pre všetky exportované funkcie"
# Výstup do súboru
$claude -H "zhrň zmeny v tomto PR" > pr-summary.md
// exit kódy

Headless mód vracia štandardné exit kódy: 0 = úspech, 1 = chyba. Pipeline môže rozhodovať podľa výsledku — napr. zablokovať merge ak agent nájde bezpečnostný problém.

02

Ako vyzerá CI/CD s Claude Code

Claude Code sa v pipeline správa ako každý iný krok — dostane kontext, vykoná úlohu a vráti výsledok. Môže čítať kód, spúšťať príkazy, písať súbory a vytvárať komentáre k PR.

📤
trigger
PR otvorený, push na main, manuálne
→
🔧
setup
Checkout, Node.js, npm install
→
🤖
claude agent
Headless úloha — review, testy, docs
→
💬
výstup
PR komentár, commit, report
03

API kľúč ako GitHub Secret

Claude Code v GitHub Actions sa autentifikuje cez API kľúč, nie cez browser login. Kľúč musí byť bezpečne uložený ako GitHub Secret — nikdy nie priamo v kóde alebo YAML súbore.

nastavenie GitHub Secret
# Pridanie Secret cez GitHub CLI
$gh secret set ANTHROPIC_API_KEY
? Enter secret value: **********************
✓ Set secret ANTHROPIC_API_KEY for MichalP777/projekt
# Alebo cez GitHub Web UI:
Settings → Secrets and variables → Actions → New repository secret
Name: ANTHROPIC_API_KEY
Value: sk-ant-api03-...
// nikdy takto

Nikdy neuvádzaj API kľúč priamo v .github/workflows/*.yml súbore — je viditeľný pre každého s prístupom k repozitáru a GitHub ho môže zneplatniť.

04

GitHub Actions workflow — príklady

Tri najčastejšie use cases s kompletným YAML:

📄 .github/workflows/claude-review.yml — automatický code review na PR
name: Claude Code Review on: pull_request: types: [opened, synchronize] jobs: review: runs-on: ubuntu-latest permissions: contents: read pull-requests: write # na písanie komentárov steps: - uses: actions/checkout@v4 with: fetch-depth: 0 # celá história pre git diff - uses: actions/setup-node@v4 with: node-version: '22' - name: Inštalácia Claude Code run: npm install -g @anthropic-ai/claude-code - name: Claude Code Review env: ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} run: | claude --headless \ "Skontroluj zmeny v tomto PR (git diff origin/main). \ Hľadaj bugy, bezpečnostné problémy a porušenie konvencií. \ Výsledok zakomentuj do PR pomocou GitHub CLI."
📄 .github/workflows/claude-tests.yml — generovanie testov pre nové súbory
name: Generate Tests on: pull_request: paths: - 'src/**/*.ts' # len keď sa zmení TypeScript kód jobs: generate-tests: runs-on: ubuntu-latest permissions: contents: write # na commitovanie nových testov steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: { node-version: '22' } - run: npm install -g @anthropic-ai/claude-code - name: Generuj testy env: ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} run: | claude --headless \ "Pre každý nový súbor v src/ ktorý nemá test, \ vytvor zodpovedajúci test súbor v tests/. \ Použi Vitest a dodržiavaj konvencie v .claude/rules/testing.md" - name: Commitni nové testy run: | git config user.name "Claude Code Bot" git config user.email "bot@anthropic.com" git add tests/ git diff --staged --quiet || git commit -m "feat: add generated tests" git push
📄 workflow_dispatch — manuálne spustenie s parametrami
name: Claude Manual Task on: workflow_dispatch: # manuálne spustenie z GitHub UI inputs: task: description: 'Úloha pre agenta' required: true default: 'Skontroluj bezpečnosť závislostí' jobs: run-task: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: npm install -g @anthropic-ai/claude-code - env: ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} run: claude --headless "${{ github.event.inputs.task }}"
05

Typické use cases v CI/CD

on: pull_request
Automatický code review
Agent skontroluje diff, nájde bugy a navrhne vylepšenia — komentár priamo na PR.
on: pull_request paths: src/**
Generovanie testov
Pre každý nový súbor bez testu agent vygeneruje test a commitne ho do PR.
on: push main
Aktualizácia dokumentácie
Po merge agent aktualizuje README, API docs alebo CHANGELOG na základe zmien.
on: schedule cron
Bezpečnostný audit
Každý týždeň agent skontroluje závislosti, permissions a potenciálne zraniteľnosti.
on: workflow_dispatch
Manuálne úlohy
Spusti agenta na demand s vlastnou úlohou — refaktoring, migrácia, analýza.
on: issue_comment
Bot na issues
Keď niekto napíše "/claude fix this", agent sa pokúsi problém vyriešiť automaticky.
06

Bezpečnosť a limity v CI/CD

best practices
✓ Vždy používaj
secrets.ANTHROPIC_API_KEY — nikdy plaintext kľúč
permissions: read len tam kde nepotrebuješ write
paths: filter — nespúšťaj agenta pre každý push
timeout-minutes: 10 — agent nesmie bežať donekonečna
⚠ Dávaj pozor na
Fork PR — externe PR môžu mať prístup k secrets
Agent commituje kód — vždy kontroluj čo commitol
Náklady — každý workflow run spotrebuje tokeny
# Ochrana fork PR — spusti len z dôveryhodných vetiev
if: github.event.pull_request.head.repo.full_name == github.repository
// fork pull requesty

GitHub Actions nezdieľa secrets s fork PR od externých prispievateľov. Ak potrebuješ agenta aj pre externé PR, použi pull_request_target trigger opatrne — má prístup k secrets, ale beží na kóde z forku.

Chceš, aby Claude Code automaticky pridal komentár ku každému novému PR s code review. Aký GitHub Actions trigger použiješ?

A on: push — spustí sa pri každom push na akúkoľvek vetvu
B on: pull_request: types: [opened, synchronize] — spustí sa pri otvorení a aktualizácii PR
C on: workflow_dispatch — manuálne spustenie z GitHub UI
D on: schedule — spustí sa v pravidelných intervaloch

Claude Code v headless móde beží autonómne — ideálny pre GitHub Actions. API kľúč ukladaj vždy ako GitHub Secret. Najčastejšie use cases: automatický code review, generovanie testov a aktualizácia dokumentácie. Vždy nastav timeout a minimálne permissions.
