Claude Code Essentials  ·  Lekcia 05

CLAUDE.md —
perzistentný kontext

CLAUDE.md je súbor, ktorý agent číta automaticky pri každom spustení. Je to tvoj spôsob, ako agentovi povedať: kto sme, čo robíme a ako to robíme.

// čo sa naučíš

  • Pochopiť, čo je CLAUDE.md a prečo ho každý projekt potrebuje
  • Vedieť, kde CLAUDE.md umiestniť — globálne, projektové, podpriečinkové
  • Napísať efektívny CLAUDE.md pre reálny projekt
  • Používať Rules súbory na jemnú kontrolu správania agenta
  • Rozoznať, čo do CLAUDE.md patrí a čo nie
01

Čo je CLAUDE.md a prečo ho potrebuješ

Keď spustíš Claude Code v priečinku projektu, agent automaticky vyhľadá a prečíta súbor CLAUDE.md. Tento súbor je trvalá inštrukcia pre agenta — platí pre každú session, každé spustenie, každého člena tímu.

Bez CLAUDE.md musíš agentovi každú session vysvetľovať kontext projektu znova. S CLAUDE.md agent hneď vie: aký je stack, aké sú konvencie, čo smie a čo nesmie.

// kedy agent číta CLAUDE.md

Pri každom spustení claude v danom priečinku — automaticky, bez toho, aby si o to žiadal. Je to prvá vec, ktorú agent načíta pred akoukoľvek tvojou správou.

môj-projekt/
├── CLAUDE.md         ← agent číta automaticky
├── src/
│   └── CLAUDE.md     ← pre src/ podpriečinok
├── tests/
├── package.json
└── .claude/
    └── settings.json
02

Kde CLAUDE.md umiestniť

Claude Code načíta viacero CLAUDE.md súborov naraz — hierarchicky. Každý súbor má iný rozsah platnosti.

~/.claude/CLAUDE.md
# Platí pre VŠETKY projekty na tvojom stroji
# Ideálne pre osobné preferencie a globálne pravidlá
Umiestnenie: ~/.claude/CLAUDE.md
Príklady obsahu:
- Preferovaný jazyk odpovedí (napr. slovenčina)
- Osobný štýl kódu
- Globálne zakázané operácie
./CLAUDE.md (koreň projektu)
# Platí pre celý projekt — commitni do git
# Zdieľaný kontext pre celý tím
Umiestnenie: ./CLAUDE.md
Príklady obsahu:
- Tech stack a architektúra projektu
- Konvencie pre kód a naming
- Testovacie príkazy a workflow
- Zakázané operácie pre daný projekt
./src/CLAUDE.md
# Platí len keď agent pracuje v danom podpriečinku
# Špecifické pravidlá pre časť projektu
Príklady:
- ./src/CLAUDE.md — pravidlá pre frontend kód
- ./api/CLAUDE.md — pravidlá pre backend / REST
- ./tests/CLAUDE.md — pravidlá pre písanie testov
// poradie načítania

Agent načítava: globálny → projektový → podpriečinkový. Neskoršie súbory dopĺňajú (nie prepísujú) predchádzajúce. Môžeš mať teda všeobecné pravidlá globálne a špecifické lokálne.

03

Čo písať do CLAUDE.md

CLAUDE.md je markdown súbor — píš ho ako inštrukcie pre nového kolegu, ktorý nepozná projekt. Agent ho číta doslova.

📄 CLAUDE.md — príklad pre webový projekt
# Claude Code — Projekt: E-shop API
## Tech Stack
Backend: Node.js 22 + Express + TypeScript
Databáza: PostgreSQL 16 + Prisma ORM
Autentifikácia: JWT + bcrypt
Testy: Vitest + Supertest
## Príkazy
### Spustenie
npm run dev — development server (port 3000)
npm run build — produkčný build
### Testy
npm test — všetky testy
npm run test:watch — watch mód
## Konvencie kódu
Funkcie: camelCase, súbory: kebab-case
Každý endpoint musí mať zodpovedajúci test
Žiadne any typy v TypeScript
Chybové hlášky vždy v angličtine
## Zakázané operácie
Nikdy nespúšťaj DROP TABLE bez potvrdenia
Nemazaj súbory v /prisma/migrations
Neupravuj .env.production
04

Čo do CLAUDE.md nepatrí

CLAUDE.md nie je miesto pre všetko. Príliš dlhý alebo zle štruktúrovaný CLAUDE.md môže agenta skôr zmiasť ako pomôcť. Drž ho stručný a relevantný.

čo kam patrí
✓ CLAUDE.md:
Architektúra a tech stack projektu
Príkazy na spustenie, build, testy
Konvencie a kódovacie štandardy tímu
Zakázané operácie a bezpečnostné pravidlá
Štruktúra priečinkov a kde čo nájsť
✗ Nepatrí do CLAUDE.md:
API kľúče a secrets (použiť .env alebo settings.local)
Celé bloky kódu (agent si ich prečíta sám)
Dokumentácia každej funkcie (to je README.md)
História projektu a changelog
Tutoriály a vysvetlenia pre ľudí
// zlaté pravidlo

CLAUDE.md by mal mať menej ako 500 riadkov. Ak je dlhší, rozdeľ ho do podpriečinkových CLAUDE.md súborov podľa oblasti (frontend, backend, testy).

05

Rules súbory — jemná kontrola správania

Okrem CLAUDE.md môžeš vytvoriť špecializované rules súbory v priečinku .claude/rules/. Tieto súbory agent načíta podľa kontextu — napríklad len pri práci s testami.

.claude/
├── settings.json
└── rules/
    ├── testing.md    ← pravidlá pre písanie testov
    ├── security.md  ← bezpečnostné pravidlá
    └── api.md      ← pravidlá pre REST API
📄 .claude/rules/testing.md
# Pravidlá pre písanie testov
Tieto pravidlá platia vždy keď píšeš alebo upravuješ testy.
## Povinné pravidlá
Každý test musí mať describe blok s názvom modulu
Používaj it() nie test() pre konzistentnosť
Mock len externé závislosti (HTTP, databáza)
Každý test musí byť izolovaný — žiadne zdieľané state
## Štruktúra testu
Arrange → Act → Assert pattern
Názov testu: "should [expected behavior] when [condition]"
// ako agent používa rules

Rules súbory môžeš referencovať aj manuálne: Pozri .claude/rules/security.md pred akoukoľvek zmenou autentifikácie. Agent ich aplikuje presne podľa inštrukcií.

06

Vytvorenie CLAUDE.md — prakticky

Claude Code ti vie pomôcť vygenerovať CLAUDE.md pre existujúci projekt:

generovanie CLAUDE.md
# Požiadaj agenta, nech sám analyzuje projekt
> Analyzuj tento projekt a vytvor CLAUDE.md s konvenciami,
  príkazmi a pravidlami, ktoré by mal agent poznať.
Analyzujem štruktúru projektu...
Čítam package.json, tsconfig.json, .eslintrc...
✓ CLAUDE.md vytvorený (87 riadkov)
# Alebo spusti automatické skenovanie
$claude --init
✓ CLAUDE.md initialized for this project
// živý dokument

CLAUDE.md nie je jednorazový — aktualizuj ho pri každej väčšej zmene projektu. Pridaj ho do code review procesu rovnako ako ostatné konfiguračné súbory.

// rýchly test

Pracuješ na projekte s frontendom a backendom. Chceš mať rôzne pravidlá pre každú časť. Ako to najlepšie zorganizuješ?

A Jeden veľký CLAUDE.md v koreni projektu so všetkým
B CLAUDE.md v koreni + samostatné CLAUDE.md v /frontend a /backend priečinkoch
C Dva samostatné projekty — jeden pre frontend, jeden pre backend
D CLAUDE.md nie je vhodný pre väčšie projekty
// zhrnutie lekcie 05

CLAUDE.md je základ každého projektu s Claude Code. Píš do neho kontext, príkazy, konvencie a zakázané operácie. Hierarchia globálny → projektový → podpriečinkový ti dáva presnosť bez duplicity. Drž ho stručný — agent ho číta pri každom spustení.
