01

Tri vrstvy nastavení

Claude Code používa hierarchický systém nastavení — každá vrstva môže prepísať tú vyššiu. Nastavenia sa ukladajú do settings.json súborov na rôznych miestach.

1. ~/.claude/settings.json Globálne — platí pre všetky projekty na tomto stroji
2. .claude/settings.json Projektové — platí pre konkrétny projekt (commit do git)
3. .claude/settings.local.json Lokálne — osobné nastavenia projektu (do .gitignore)
// best practice

Zdieľaj settings.json cez git pre tímové nastavenia. Do settings.local.json ukladaj osobné preferencie a API kľúče — pridaj ho do .gitignore.

settings.json — základná štruktúra
{
  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Bash(npm:*)", "WebFetch"],
    "deny": ["Bash(rm:*)"]
  },
  "vim": false
}
02

Permission Rules — čo agent smie robiť

Permission rules určujú, ktoré nástroje a príkazy agent môže vykonať bez pýtania sa. Fungujú na princípe allow-list a deny-list.

Bash oprávnenia
Bash(npm:*) — všetky npm príkazy
Bash(git:*) — všetky git príkazy
Bash(ls:*) — výpisy priečinkov
Bash(cat:/src/*) — čítanie v /src
Sieťové oprávnenia
WebFetch — sťahovanie obsahu
WebFetch(domain:*.github.com)
mcp__playwright — ovládanie browsera
mcp__* — všetky MCP nástroje
Deny — zakázané
Bash(rm:*) — mazanie súborov
Bash(sudo:*) — root príkazy
Bash(curl:*) — sieťové volania
Bash(chmod:*) — zmena práv
nastavenie cez CLI
# Povoliť npm príkazy globálne
$claude config set permissions.allow '["Bash(npm:*)"]'
✓ Permission added: Bash(npm:*)
# Zakázať mazanie súborov
$claude config set permissions.deny '["Bash(rm:*)"]'
✓ Permission denied: Bash(rm:*)
# Zobraziť aktuálne nastavenia
$claude config list
model: claude-sonnet-4-6
permissions.allow: Bash(npm:*), WebFetch
permissions.deny: Bash(rm:*)
// princíp minimálnych práv

Vždy začni s deny-all prístupom a pridávaj len to, čo agent skutočne potrebuje. Je ľahšie pridať oprávnenie neskôr, ako riešiť škody po nechcenom príkaze.

03

Sandboxing — bezpečné a nebezpečné scenáre

Sandboxing obmedzuje agenta na konkrétny priečinok alebo sadu príkazov. Správne nastavený sandbox ti dáva slobodu nechať agenta pracovať autonómne bez strachu z nechcených vedľajších účinkov.

// bezpečné scenáre
Agent má prístup len do /src priečinka
Bash povolený len pre npm test a npm build
WebFetch obmedzený na dôveryhodné domény
Git príkazy povolené len ako read-only (git log, git diff)
Dev container izoluje agenta od hostiteľa
// nebezpečné scenáre
Sandbox vypnutý v produkčnom prostredí
Agent má prístup k ~/.ssh alebo ~/.aws
Bash(sudo:*) povolený bez obmedzenia
WebFetch bez whitelist — agent môže volať ľubovoľné URL
API kľúče priamo v settings.json (nie .local)
// bezpečnostné upozornenie

Nikdy nespúšťaj Claude Code s plnými oprávneniami v produkčnom prostredí. Ak agent dostane prístup k SSH kľúčom alebo AWS credentials, môže neúmyselne spôsobiť vážne škody.

bezpečná konfigurácia projektu — .claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(npm:*)",
      "Bash(git log:*)",
      "Bash(git diff:*)",
      "WebFetch(domain:*.npmjs.com)"
    ],
    "deny": [
      "Bash(rm:*)",
      "Bash(sudo:*)",
      "Bash(curl:*)"
    ]
  }
}
04

VIM mode a výber modelu

Claude Code CLI podporuje VIM keybindings pre tých, ktorí preferujú modálne editovanie. Okrem toho môžeš kedykoľvek prepnúť model, ktorý agent používa.

VIM mode a model
# Zapnúť VIM mode
$claude config set vim true
✓ VIM mode enabled
# Vypnúť VIM mode
$claude config set vim false
# Nastaviť model pre projekt
$claude config set model claude-opus-4-6
✓ Model set: claude-opus-4-6
# Jednorazová zmena modelu pre session
$claude --model claude-haiku-4-5 "rýchla úloha"
# Prepínanie modelu počas session
> /model claude-sonnet-4-6
✓ Switched to claude-sonnet-4-6
// výber modelu podľa úlohy

Haiku — rýchle jednoduché úlohy, nízka cena  ·  Sonnet — ideálny pre väčšinu kódovania  ·  Opus — komplexné architektonické rozhodnutia

05

Permission módy — interaktívny vs. autonómny

Claude Code má niekoľko módov, ktoré určujú, ako agresívne sa pýta na potvrdenie pred vykonaním akcií. Vyber si podľa toho, aký stupeň kontroly chceš mať.

permission módy
# Predvolený mód — pýta sa pred každou akciou
$claude
# Autonómny mód — vykoná akcie bez pýtania (opatrne!)
$claude --dangerously-skip-permissions
⚠ All permission checks disabled. Use with caution.
# Headless mód — pre CI/CD (bez interakcie)
$claude --headless "spusti testy a oprav chyby"
# Počas session — zmena módu schvaľovania
> /auto-approve bash
✓ Bash commands will be auto-approved this session
// --dangerously-skip-permissions

Tento flag obchádza všetky bezpečnostné kontroly. Nikdy ho nepoužívaj v produkčnom prostredí ani na systéme s citlivými dátami. Určený výlučne pre izolované sandboxové prostredia.

// rýchly test

Kde uložíš osobný API kľúč tak, aby sa nedostal do git repozitára, ale stále fungoval v projekte?

A ~/.claude/settings.json — globálne nastavenia
B .claude/settings.json — projektové nastavenia (commitnuté do git)
C .claude/settings.local.json — lokálne nastavenia (v .gitignore)
D Priamo do CLAUDE.md súboru projektu
// zhrnutie lekcie 04

Nastavenia majú tri vrstvy — globálne, projektové a lokálne. Permission rules riadia, čo agent smie vykonať bez pýtania. Vždy používaj princíp minimálnych práv a citlivé údaje ukladaj do settings.local.json mimo git.
